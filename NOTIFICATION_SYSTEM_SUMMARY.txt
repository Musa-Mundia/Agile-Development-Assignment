═══════════════════════════════════════════════════════════════════
  📋 HOW IT WORKS
═══════════════════════════════════════════════════════════════════

ADDING NOTIFICATIONS:
  window.XGymNotifications.addNotification('messenger', 1);
  window.XGymNotifications.addNotification('workouts', 2);

CLEARING NOTIFICATIONS:
  window.XGymNotifications.clearNotification('messenger');

AUTO-CLEAR:
  - When user clicks on navigation item
  - 500ms delay to allow smooth panel transition
  - Automatically mapped to correct notification type

STORAGE:
  - localStorage (persists across refresh)
  - Separate keys for each dashboard type
  - JSON format for easy inspection

═══════════════════════════════════════════════════════════════════
  🔗 INTEGRATION WITH BackendMain.js
═══════════════════════════════════════════════════════════════════

Example for Client receiving new workout:
  firebase.database().ref('workouts/' + clientId)
    .on('child_added', (snapshot) => {
      window.XGymNotifications.addNotification('workouts', 1);
    });

Example for Trainer receiving new message:
  firebase.database().ref('messages/' + trainerId)
    .on('child_added', (snapshot) => {
      window.XGymNotifications.addNotification('messenger', 1);
    });

Example for Admin - new member:
  firebase.database().ref('members')
    .on('child_added', (snapshot) => {
      window.XGymNotifications.addNotification('members', 1);
    });
